DESCRIPTION

v.interp.timeseries interpolates a station-based climate time series (stored in a SQLite table by v.in.ghcn or a similar module) to raster grids, to point samples, or to polygon areas.

Three interpolation methods are provided: inverse-distance weighting (method=idw), thin-plate spline (method=spline), and covariate regression with spline residuals (method=regression).

NOTES

Inputs

The station vector map (input) and its companion SQLite table (table) are typically produced by v.in.ghcn. The table must contain at minimum the columns cat, datetime, element, and value as written by that module. By default table is {input}_timeseries.

Methods

method=AlgorithmNotes
idw Inverse-distance weighting Controlled by power and npoints. Pure numpy + scipy KDTree; fast.
spline Thin-plate spline (RBF) Exact interpolator via scipy.interpolate.RBFInterpolator. May produce artefacts in extrapolation; use -m to mask.
regression Linear regression on covariates + thin-plate spline on residuals Requires covariates=. Analogous to PRISM-style detrended interpolation. Falls back to plain spline if no covariates are supplied.

Covariates

When method=regression, one or more raster maps are supplied via covariates= (comma-separated, e.g. dem,slope). These must exist in the current mapset or be visible across mapsets. A least-squares regression is fit between station values and the covariate values at station locations; the residuals are then interpolated by thin-plate spline and added back to the predicted surface.

Elevation is the most common covariate: temperature decreases and orographic precipitation increases with altitude. Any raster may be used, however — topographic slope, aspect, distance to coast, etc.

Output

When output= is supplied, one FCELL raster map is written per time step, named <output>_YYYYMMDD (daily) or <output>_YYYYMM01 (monthly). Use -t to register these automatically as a GRASS space-time raster dataset (strds).

Sample extraction

When sample= is supplied, interpolated values are extracted at the sample locations and written to a SQLite table named <sample>_timeseries in the mapset database.

sample= geometryExtraction method
Points Nearest-cell lookup from the interpolated grid (or direct interpolation at point coordinates if output= is omitted)
Areas (polygons) Area-weighted mean of all grid cells within each polygon. Requires output=. Results are keyed by polygon cat and conceptually linked to the polygon centroid.

Missing data and fill

Time steps with fewer than min_stations stations reporting data are skipped. With -f, stations that are missing on a given day borrow their value from the nearest station that does have data. This keeps the spatial coverage consistent across the time series but introduces spatial smoothing; use with care for sparse networks.

Convex hull mask

With -m, output raster cells outside the convex hull of the active stations are set to null. This prevents extrapolation artefacts, particularly for the spline method. The mask is recomputed for each time step because the set of reporting stations can vary.

Error metric

When output= is given, leave-one-out (LOO) cross-validation is performed at each time step and the root-mean-square error (RMSE) is stored in a SQLite table named <output>_errors: 

datetime   TEXT      -- ISO 8601 date
element    TEXT      -- e.g. PRCP
rmse       REAL      -- LOO cross-validation RMSE (NULL if not computed)
n_stations INTEGER   -- number of stations used

LOO is computed before neighbor fill so it reflects actual data density. For method=spline with more than 50 active stations the LOO step is skipped (thin-plate spline requires O(n³) work per fold) and RMSE is stored as NULL.

Sample table schema

cat        INTEGER   -- GRASS category (polygon or point)
datetime   TEXT      -- ISO 8601 date
element    TEXT      -- e.g. PRCP
value      REAL      -- interpolated value in source units (mm, °C, …)

EXAMPLES

Interpolate PRCP to a daily raster grid using IDW, register as a strds: 
v.interp.timeseries input=gauges element=PRCP method=idw output=prcp -t
Spline interpolation with convex hull masking, 2000–2020: 
v.interp.timeseries input=gauges element=PRCP method=spline output=prcp \
    start_date=2000-01-01 end_date=2020-12-31 -t -m
Regression against a DEM with neighbor fill for missing stations: 
v.interp.timeseries input=gauges element=TMAX method=regression \
    covariates=dem output=tmax start_date=1990-01-01 -t -f
Extract time series at watershed centroids only (no grid): 
v.interp.timeseries input=gauges element=PRCP method=idw \
    sample=watersheds start_date=2010-01-01
Produce both grid output and watershed-averaged time series: 
v.interp.timeseries input=gauges element=PRCP method=regression \
    covariates=dem output=prcp sample=watersheds -t -m
Query the sample table: 
db.select sql="SELECT cat, datetime, value FROM watersheds_timeseries \
  WHERE cat=1 ORDER BY datetime"

# Annual totals per watershed
db.select sql="SELECT cat, substr(datetime,1,4) AS year, SUM(value) AS prcp_mm \
  FROM watersheds_timeseries WHERE element='PRCP' GROUP BY cat, year"
Query the error table: 
db.select sql="SELECT datetime, rmse, n_stations FROM prcp_errors ORDER BY datetime"
